# MCP Builder Skill

Generate fully functional Model Context Protocol (MCP) servers from natural language specifications or existing API definitions.

## What is MCP?

The [Model Context Protocol](https://modelcontextprotocol.io) is an open standard that lets AI models interact with external tools and data sources through a uniform interface. An MCP server exposes:
- **Tools** — callable functions the model can invoke (e.g., `search_database`, `send_email`)
- **Resources** — readable artifacts the model can access (e.g., `file://data.csv`, `db://customers`)
- **Prompts** — reusable prompt templates the model can reference

## Transports

| Transport | Use case |
|-----------|----------|
| `stdio` | CLI tools, local processes (default) |
| `SSE` | Long-lived HTTP streams (web servers, remote agents) |
| `WebSocket` | Bidirectional real-time (browser clients) |

## Python MCP Server Template

```python
#!/usr/bin/env python3
"""
{server_name} — MCP server generated by Pakalon mcp-builder skill.
Usage: python server.py          # stdio transport (default)
       python server.py --sse    # SSE transport on port 8080
"""
from __future__ import annotations

import argparse
import asyncio
import json
from typing import Any

from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp.server.sse import SseServerTransport
from mcp.types import (
    CallToolRequest,
    CallToolResult,
    ListToolsRequest,
    ListToolsResult,
    TextContent,
    Tool,
)

app = Server("{server_name}")


@app.list_tools()
async def list_tools() -> list[Tool]:
    return [
        Tool(
            name="example_tool",
            description="An example tool that echoes its input.",
            inputSchema={
                "type": "object",
                "properties": {
                    "message": {"type": "string", "description": "Message to echo"},
                },
                "required": ["message"],
            },
        ),
    ]


@app.call_tool()
async def call_tool(name: str, arguments: dict[str, Any]) -> list[TextContent]:
    if name == "example_tool":
        return [TextContent(type="text", text=f"Echo: {arguments['message']}")]
    raise ValueError(f"Unknown tool: {name}")


async def main(use_sse: bool = False, port: int = 8080) -> None:
    if use_sse:
        from starlette.applications import Starlette
        from starlette.routing import Route

        transport = SseServerTransport("/messages")

        async def handle_sse(request):
            async with transport.connect_sse(request.scope, request.receive, request._send) as streams:
                await app.run(streams[0], streams[1], app.create_initialization_options())

        starlette_app = Starlette(routes=[Route("/sse", endpoint=handle_sse)])
        import uvicorn
        await uvicorn.Server(uvicorn.Config(starlette_app, host="0.0.0.0", port=port)).serve()
    else:
        async with stdio_server() as (read, write):
            await app.run(read, write, app.create_initialization_options())


if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument("--sse", action="store_true")
    parser.add_argument("--port", type=int, default=8080)
    args = parser.parse_args()
    asyncio.run(main(use_sse=args.sse, port=args.port))
```

## Generating an MCP Server from an OpenAPI Spec

```python
def openapi_to_mcp_tools(openapi_spec: dict) -> list[dict]:
    """Convert OpenAPI paths to MCP Tool definitions."""
    tools = []
    for path, methods in openapi_spec.get("paths", {}).items():
        for method, op in methods.items():
            if method.upper() not in ("GET", "POST", "PUT", "DELETE", "PATCH"):
                continue
            operation_id = op.get("operationId") or f"{method}_{path.replace('/', '_').strip('_')}"
            params = op.get("parameters", [])
            body_schema = (op.get("requestBody", {}).get("content", {})
                           .get("application/json", {}).get("schema", {}))

            properties = {p["name"]: {"type": p.get("schema", {}).get("type", "string"),
                                       "description": p.get("description", "")}
                          for p in params}
            if body_schema.get("properties"):
                properties.update(body_schema["properties"])

            tools.append({
                "name": operation_id,
                "description": op.get("summary", f"{method.upper()} {path}"),
                "inputSchema": {
                    "type": "object",
                    "properties": properties,
                    "required": [p["name"] for p in params if p.get("required")],
                },
                "_meta": {"method": method.upper(), "path": path},
            })
    return tools
```

## Generation Workflow

1. Parse the specification (natural language → structured requirements).
2. Enumerate all tools with names, descriptions, and JSON Schema `inputSchema`.
3. Map each tool to its implementation (API call, DB query, file operation, etc.).
4. Emit the Python file using the template above.
5. Add `pyproject.toml` / `requirements.txt` with `mcp>=1.0.0` dependency.
6. Register in `~/.pakalon/mcp-servers.json`.

## Best Practices

- Use `snake_case` for tool names — they must match identifiers in Claude's tool calls.
- Always validate required parameters before making side-effecting calls.
- Return structured JSON inside `TextContent.text` for machine-readable output.
- Prefer stdio transport for local tools; SSE for hosted/remote tools.
- Include a `--help` flag so users can discover the server's capabilities without Claude.
